## 2. 产品概述

### 2.1 产品定位与目标用户

**Echo Day** 是一款以**情绪和记忆**为核心的 AI 日记应用。名字取自两层含义：音乐的回响（Echo）和每日的记录（Day）。通过**照片 + AI 语音对话**的组合方式，解决用户"想记录但不愿动笔"的核心痛点，并借助情绪分析与音乐匹配，实现视觉、文字、听觉三维度的沉浸式记忆回溯体验。

**目标用户：** 有记录生活意愿但懒得动笔，或希望日后重温某天情绪氛围的年轻用户。

### 2.2 核心价值主张

1. **零门槛记录：** 语音对话替代手动写作，AI 自动生成完整日记。
2. **情绪驱动的音乐匹配：** AI 理解对话与照片氛围，推荐最契合当日情绪的曲目。
3. **沉浸式记忆回溯：** 照片、日记文字、音乐同时呈现，三维度重现当日氛围。
4. **情绪日历：** AI 自动推测当日心情并以图标呈现在日历中，情绪变化一目了然。

## 3. 核心功能模块

### 3.1 每日录制向导

#### 3.1.1 照片采集与草稿创建

| 功能点 | 描述 |
| :--- | :--- |
| **拍摄/上传** | 从本地上传照片，每天仅限一张，支持预览和重选。**照片为必填项**，未上传照片不能进入对话步骤 |
| **云端存储** | 上传到 Supabase Storage，获取持久化路径 |
| **草稿即时创建** | 照片上传完成后立即创建 DRAFT 记录，支持中途退出后断点续接 |
| **每日冲突处理** | 若当日已有完成的记录，"Record Today"按钮变为"重新记录"；点击后弹出二次确认，告知将覆盖今天已有的日记、音乐和心情；如仅需修改内容，可从主页点进详情页编辑 |

#### 3.1.2 AI 语音对话

用户自然讲述，AI 适时追问，模拟与朋友倾诉的体验。语音对话由三个模型协作完成：

- **语音转文字：** gpt-4o-mini-transcribe 将用户语音转为文字
- **对话生成：** gpt-5-mini 根据对话上下文生成回复内容
- **文字转语音：** gpt-4o-mini-tts 将 AI 回复朗读给用户

对话过程中同步生成完整的聊天文本记录（transcript），供后续日记生成使用。

| 功能点 | 描述 |
| :--- | :--- |
| **语音输入** | 点击麦克风按钮开始录音，再次点击停止并发送。AI 回复以语音朗读 + 文字同步显示 |
| **打字输入** | 作为语音的静音替代方案：用户打字发送，AI 回复仅显示文字，不触发 TTS 朗读 |
| **实时字幕** | 对话过程中同步显示双方的文字内容 |
| **结束对话** | 用户主动结束，完整 transcript 传递至下一步 |

**AI 对话策略：** AI 以温和的好奇心引导叙述，聚焦"今天发生了什么"和"你当时的感受"，不做评判、不提建议，只倾听和追问。

#### 3.1.3 AI 日记生成与编辑

对话结束后触发一次 AI 生成请求，基于 transcript + 照片，一次性生成日记、情绪分析和音乐推荐。

| 功能点 | 描述 |
| :--- | :--- |
| **AI 生成日记** | 基于对话转录 + 照片内容生成约 150–220 字第一人称日记，SSE 流式输出 |
| **情绪分析** | AI 同时输出 valence 和 arousal 两个连续值，系统映射为离散情绪 |
| **音乐推荐** | AI 输出 Spotify 搜索词 + 推荐理由，前端调用 Spotify Search API 获取曲目 |
| **可编辑预览** | 用户可直接修改 AI 生成的日记文字 |

#### 3.1.4 音乐确认

| 功能点 | 描述 |
| :--- | :--- |
| **展示 AI 推荐曲目** | 呈现封面图、曲名、歌手 |
| **30s 预听** | 播放 Spotify 提供的试听片段，若无 previewUrl 则隐藏播放按钮 |
| **确认这首** | 用户接受推荐 → 保存记录 |
| **换一首** | 从当前 Spotify 搜索结果列表中切换另一首，无额外 API 调用 |
| **重新推荐** | 列表中都不满意时，重新调用 AI 生成新的搜索词，再调 Spotify 获取新结果 |

#### 3.1.5 情绪系统定义

AI 输出两个连续值描述当日情绪：

- **valence**（效价，0–1）：情绪的正负面程度。0 为最负面，1 为最正面。
- **arousal**（唤醒度，0–1）：情绪的能量强度。0 为最平静/低能量，1 为最激动/高能量。

两个值组成一个二维坐标，系统将其划分为 3×3 网格，映射到 9 种离散情绪：

| | 低 valence (0–0.33) | 中 valence (0.33–0.66) | 高 valence (0.66–1) |
| :--- | :--- | :--- | :--- |
| **高 arousal (0.66–1)** | 愤怒 | 惊讶 | 兴奋 |
| **中 arousal (0.33–0.66)** | 悲伤 | 平淡 | 愉悦 |
| **低 arousal (0–0.33)** | 低落 | 疲惫 | 平静 |

映射逻辑在 `lib/mood-map.ts` 中实现，AI 只负责输出数值，不参与情绪分类。用户可在详情页手动修改情绪，修改后原始 valence/arousal 数值保留不变。

### 3.5 应用流程设计

整体流程分为两个阶段：**语音对话阶段**（Step 1–5 循环进行）和**内容生成阶段**（Step 6–7 一次性完成）。

#### 阶段一：语音对话

用户上传照片后进入对话页面，以下步骤循环进行，直到用户主动结束：

| Step | 描述 | 调用模型 |
| :--- | :--- | :--- |
| 1 | 用户点击说话按钮，前端开始录音 | — |
| 2 | 语音转文字，转录结果追加到聊天记录 | gpt-4o-mini-transcribe |
| 3 | 基于完整聊天上下文生成 AI 回复，回复文本追加到聊天记录 | gpt-5-mini |
| 4 | 将 AI 回复朗读给用户 | gpt-4o-mini-tts |
| 5 | 用户继续说话 → 回到 Step 1；用户主动结束对话 → 进入阶段二 | — |

#### 阶段二：内容生成

| Step | 描述 | 调用模型 |
| :--- | :--- | :--- |
| 6 | 收集完整聊天文本 + 用户上传的照片，单次请求生成：日记文本、歌曲推荐（搜索词 + 推荐理由）、情绪分析（valence + arousal） | gpt-5-mini |
| 7 | 拿到歌曲搜索词后调用 Spotify Search API，返回匹配曲目 | — |

### 4.3 页面结构

应用共 3 个核心页面：

#### 1. 主页（Home）

日历视图与时间轴视图切换展示所有历史记录。

- 顶部：欢迎语 + Calendar / Timeline 视图切换
- 日历视图：月份网格，有记录的日期显示心情图标，支持前后翻月
- 时间轴视图：按日期倒序的卡片流，虚拟滚动 + 懒加载分页
- 点击某一天 → 进入日记详情页
- 底部固定"Record Today"按钮 → 进入记录页

#### 2. 记录页（Record）

用户在此完成照片上传和 AI 语音对话。

- 上传照片：从相册选取，仅限一张，支持预览和重选
- 语音对话：麦克风按钮（按住/点击说话），实时显示双方文字字幕，支持语音或打字输入
- 结束按钮：用户主动结束对话，携带完整 transcript + 照片跳转至详情页，触发内容生成

#### 3. 日记详情页（Detail）

展示与编辑合一，首次生成和回访共用同一页面。

**首次生成（从记录页跳转）：**
- 直接进入页面，音乐立即播放（记录页的交互已满足浏览器手势要求）
- 日记文本流式生成，实时渲染
- 音乐卡片展示 AI 推荐曲目
- 用户可编辑日记文本、换一首音乐、修改心情
- 点击完成 → 保存记录

**回访（从主页日历/时间轴点入）：**
- 先展示入口遮罩：全屏照片 + 心情关联色蒙层 + "进入这一天"按钮
- 点击按钮后遮罩退出，音乐开始播放，展示完整日记内容
- 同样可编辑日记、换音乐、修改心情

### 4.4 后端接口与数据库设计

#### 4.4.1 API 接口定义

所有接口均为 Next.js Route Handler（`app/api/` 目录），运行在服务端。

**认证**

| 接口 | 方法 | 描述 |
| :--- | :--- | :--- |
| `/api/auth/[...nextauth]` | — | NextAuth.js 处理 Google OAuth 登录/注册 |

**照片**

| 接口 | 方法 | 描述 |
| :--- | :--- | :--- |
| `/api/photos/upload` | POST | 接收单张照片，校验类型和大小（≤10MB），上传到 Supabase Storage，返回 `{ photoKey, photoUrl }` |

**日记记录**

| 接口 | 方法 | 描述 |
| :--- | :--- | :--- |
| `/api/entries` | GET | 获取记录列表。支持查询参数：`year` + `month`（按月查询，日历翻页用）、`mood`（按心情筛选，可选）。返回最小投影：date, moodEmoji, moodColorHex, photoUrl |
| `/api/entries` | POST | 创建 DRAFT 记录（照片上传完成后调用），返回 `entryId` |
| `/api/entries/[id]` | GET | 获取某天完整日记详情 |
| `/api/entries/[id]` | PATCH | 更新记录（日记文本 / 音乐 / 心情 / status） |
| `/api/entries/[id]` | DELETE | 删除记录，同时清理 Storage 中的照片 |
| `/api/entries/[id]/mood` | PATCH | 用户手动修改心情，更新 moodEmoji + moodColorHex + moodEmojiSource，保留原始 valence/arousal |
| `/api/entries/today` | GET | 检查当日是否已有记录，返回记录状态（无记录 / DRAFT / COMPLETE），用于主页按钮显示逻辑 |

**AI 相关**

| 接口 | 方法 | 描述 |
| :--- | :--- | :--- |
| `/api/ai/transcribe` | POST | 接收音频文件，调用 gpt-4o-mini-transcribe，返回 transcript 文本 |
| `/api/ai/chat` | POST | 接收聊天记录上下文，调用 gpt-5-mini，返回 AI 回复文本（可流式） |
| `/api/ai/tts` | POST | 接收文本，调用 gpt-4o-mini-tts，返回音频流 |
| `/api/ai/generate` | POST | 接收完整 transcript + photoUrl，调用 gpt-5-mini，SSE 流式返回日记文本，流末尾返回结构化 JSON（emotion + music） |

**音乐**

| 接口 | 方法 | 描述 |
| :--- | :--- | :--- |
| `/api/spotify/search` | POST | 接收搜索词，使用 Client Credentials Flow 调用 Spotify Search API，返回曲目列表（含 previewUrl） |

## 5. 商业模式与合规

### 5.1 权限与付费模式

| 版本 | 核心权益 | 限制 | 定价 |
| :--- | :--- | :--- | :--- |
| **免费版** | 基础录制向导；日历 + 时间轴浏览；30s 音乐预听 | 每日 1 次对话，单次对话上限约 3 分钟 | 免费 |
| **Pro 版** | 无限制对话时长与次数；历史记录数据导出 | 无限制 | $4.99/月 或 $39.99/年 |

免费用户每日对话限制（约 3 分钟）足够完成一篇日记，同时将单用户每日 API 成本控制在 ~$0.02–0.05。

### 5.2 数据隐私

用户的 transcript、照片、情绪数据存储在 Supabase 数据库中，属于敏感个人数据。隐私策略需要明确：

- 数据仅用于生成日记和情绪分析，不用于任何模型训练
- OpenAI API 默认不会将 API 输入用于模型训练
- 用户可随时删除单条记录或注销账户，删除所有关联数据（包括数据库记录和 Storage 照片）
- 图片上传校验文件类型与大小上限（10MB），防止存储滥用

---

## 6. 交互设计要点

1. **心情图标作为情绪索引：** 日历每格展示当日心情图标，一眼扫过一个月的情绪变化；详情页可手动修改。
2. **物理连续的页面过渡：** 从日历进入详情页时，心情图标通过 Framer Motion `layoutId` 平滑过渡，营造"走进那一天"的沉浸感。
3. **入口遮罩的双重作用：** 详情页的全屏入口遮罩既是仪式感设计，也是解决浏览器 autoplay 策略的工程方案。
4. **向导步骤的进度反馈：** 录制向导顶部显示步骤进度指示器，让用户清楚当前位置和剩余步骤。

---

## 7. 设计风格

**极简 + 情绪感。** 界面元素尽可能少，留白充足，让照片和文字成为唯一的视觉焦点。情绪通过色调蒙层、动画节奏和音乐氛围传递，而非堆砌装饰。

1. **照片优先：** 排版以照片为视觉中心，文字为照片的情感注脚，UI 元素退到最低存在感。
2. **情绪色调克制：** 心情关联色仅作为照片上的微弱蒙层，不喧宾夺主，整体色彩保持中性。
3. **呼吸感排版：** 日记文字使用舒适行高与字间距，大量留白，突出沉浸式阅读体验。
4. **流畅过渡：** 页面切换均有 Framer Motion 动画，节奏平缓，营造"慢慢走进回忆"的感受。

## 8. 后续优化方向

- **语义搜索：** 引入 Supabase pgvector 扩展，将日记文本做 embedding 存储，支持用户以自然语言搜索历史日记（如"找到那天我去海边很开心的日记"）。当前版本所有查询均为按日期的结构化查询，暂不需要矢量库。
